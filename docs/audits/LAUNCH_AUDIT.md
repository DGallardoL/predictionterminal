## 1. Security

### 1.1 Hardcoded secrets / API keys

**Result**: Clean.

`grep -rEn "(api_key|password|secret|token)\\s*=\\s*['\\\"][A-Za-z0-9_\\-]{16,}"
api/src/pfm/` returns **zero** hits after filtering tests/examples/TODOs.
The single admin-token persisted to disk is autogenerated on first boot
with `secrets.token_urlsafe` and stored at `/tmp/pfm_admin_token.json`
with chmod 0600 (`api/src/pfm/auth/production.py:50` and surrounding
lines). The contract is documented in
`docs/PRODUCTION_CHECKLIST.md:11-12`.

`.env.example` (root) ships placeholders only: `SENTRY_DSN=`,
`SLACK_WEBHOOK_URL=`, `PFM_METRICS_TOKEN=` etc. `.env` is gitignored
(`.gitignore:.env / .env.*`) and `git ls-files | grep .env` returns no
real `.env` tracked.

**Severity**: clean. No finding.

### 1.2 Dangerous code-execution patterns

`os.system` is **never used**. `subprocess.run` / `subprocess.Popen`
appear in three locations, all with fixed argv (no shell, no
user-controlled string):

| File:line                                  | Use                                          | Risk |
|--------------------------------------------|----------------------------------------------|------|
| `api/src/pfm/health_router.py:36`          | `git rev-parse --short HEAD` (capped 2 s)    | none |
| `api/src/pfm/main.py:550`                  | `pgrep -f arb_engine.py` (boot only)         | none |
| `api/src/pfm/main.py:585`                  | spawn `python arb_engine.py --mode <env>`    | LOW  |

The arb-engine Popen path (`main.py:585`) takes its `--mode` value from
the `PFM_ARB_ENGINE_MODE` env var (`os.environ.get(..., "og")`).
Operator-controlled, not user-controlled, but worth documenting:

**Finding S-1 (LOW)** ŌĆö `api/src/pfm/main.py:585-595`. The arb engine
mode argument is read from an env var and passed unsanitised to argv. In
practice this is fine (argv, not shell, and env vars are operator
trusted) but a defensive whitelist (`if env_mode not in {"og", "binary",
"calendar"}: env_mode = "og"`) would make the intent explicit.
**Fix**: add a `_VALID_MODES = {...}` set and `assert` at the top of the
spawn block.

`eval` appears **only** as `redis.eval(LUA, ...)` ŌĆö the Lua-script
release primitive for distributed locks (`redis_lock.py:176, 197` and
seven sites in `main.py`). This is the correct API for atomic Redis
CAS-release and the Lua scripts are constants defined in the module
header (not user input). **No security risk.**

`exec` (Python builtin) appears **zero** times.

### 1.3 Unsafe deserialization (pickle)

Five pickle.load / loads call-sites:

| File:line                                                   | Source        | Trust  |
|-------------------------------------------------------------|---------------|--------|
| `api/src/pfm/cache_pool.py:203`                             | Redis L2      | trust  |
| `api/src/pfm/terminal/__init__.py:135`                      | Redis L2      | trust  |
| `api/src/pfm/terminal/__init__.py:414`                      | local pickle  | trust  |
| `api/src/pfm/terminal/factor_clusters.py:170`               | local pickle  | trust  |
| `api/src/pfm/terminal/vol_distribution.py:121`              | local pickle  | trust  |

The first two are gated by a `PFMCP1` magic header + version envelope
(`cache_pool.py:201-206`), documented in `docs/adrs/ADR-0013-pickle-
versioning.md`. The remaining three load from on-disk prewarm files
written by our own prewarm jobs ŌĆö trusted, but the failure mode (a
corrupt cache file) is silently swallowed.

**Finding S-2 (MED)** ŌĆö Three terminal pickle.load sites operate on
local files. Threat model is "operator trust", but if an attacker can
write to `web/data/*.pkl` or `terminal/data/*.pkl` they get arbitrary
code exec.
**Fix**: (a) document that the cache dir must be owned by the api user
and not world-writable; (b) consider switching to `joblib` with
`allow_pickle=False` or pure-JSON envelopes for the three on-disk
caches. None of them store rich types ŌĆö they're all `dict[str, float]`
or `pd.DataFrame.to_dict()` ŌĆö so a JSON serializer would be a one-line
swap and would eliminate the pickle-RCE surface entirely.

### 1.4 SQL injection

The project does not use a SQL database for its core data path. The
alerts module uses SQLite via SQLAlchemy ORM only. **No SQL-injection
surface.**

### 1.5 CORS posture

`api/src/pfm/main.py:1438-1684` ŌĆö CORS is **opt-in by env var**:

```
_cors_raw = os.environ.get("PFM_CORS_ORIGINS") or os.environ.get("CORS_ORIGINS")
            or "http://127.0.0.1:8080,http://localhost:8080"
```

The dev default is **NOT wildcard**. `allow_credentials` is auto-disabled
if any origin is `*` (`main.py:1447`). CORS is registered LAST so it is
the outermost middleware (intentional, documented at `main.py:1670-
1697`), guaranteeing the `Access-Control-Allow-Origin` header is present
on 401 / 429 / 500 responses too. `docs/PRODUCTION_CHECKLIST.md:14-16`
already gates the production launch on `CORS_ORIGINS=https://your-
domain` (no wildcard). **Posture is correct.** No finding.

### 1.6 Auth

`api/src/pfm/auth/production.py` flips auth ON automatically on Fly.io /
Render / `ENV=production`. If `PFM_ADMIN_TOKEN` is unset, a strong-
randomness token is generated, persisted at `/tmp/pfm_admin_token.json`
chmod 0600, and logged once. The `is_auth_enabled()` helper centralises
the gate (`auth/__init__.py:13`). `RateLimitMiddleware` registers
unconditionally but no-ops when auth is off (`auth/rate_limiter.py:205`).
**Posture is good.**

### 1.7 Other security

- No `print()` in production paths (manual scan; CLAUDE.md mandates
  `logging` only).
- No `random.random()` used for security primitives ŌĆö `secrets` is used
  for tokens.
- `httpx` is the HTTP client everywhere; **no `requests.*` calls**
  (`grep` returns zero) ŌĆö TLS is on by default.
- Webhooks: HMAC verification is gated by `PFM_ALERTS_HMAC_SECRET`
  per `docs/PRODUCTION_CHECKLIST.md:23`.

**Finding P-1 (LOW)** ŌĆö Spot-check the rest of the 271 endpoints for

### 2.4 Missing caches on hot paths

`docs/adrs/0004-redis-cache-ttl.md` + `ADR-0008-cache-tiering.md` cover

PM-VIX, earnings dashboard, live signals, factor history. Hot-path
coverage is good.

**No finding.**

### 2.5 Memory leaks (unbounded dicts)

`grep -rEn "^_[a-z_]+\\s*:\\s*dict\\[" api/src/pfm/` finds 5 module-
level dicts. Each was reviewed:

| File:line                                          | Bound by                  | Risk |
|----------------------------------------------------|---------------------------|------|
| `cache_utils.py:161`                               | named-instance enum (~10) | none |
| `crypto5min/router.py:63`                          | TTL'd + small market set  | none |

**Finding P-2 (LOW)** ŌĆö `crypto5min/strike_scraper.py:73`

loop or LRU-cap to 10 000.
**Fix**: wrap in a `cachetools.LRUCache(maxsize=10_000)` or add a TTL
sweep at strike-scraper init.

## 3. Reliability

### 3.1 Endpoints without timeouts

`grep -rEn "httpx\\.AsyncClient\\(\\)" api/src/pfm/` returns one
non-pool site:

**Finding R-1 (MED)** ŌĆö `api/src/pfm/terminal/bulk_export.py:253` ŌĆö
`httpx.AsyncClient()` is constructed without `timeout=...`. httpx's
default is **5 s connect + 5 s read** ŌĆö which is fine ŌĆö but every other
client-construction in the codebase passes an explicit `timeout=` so the
behaviour is documented at the call site. The fan-out is parallel via
`asyncio.gather` over `req.slugs`; if the caller submits 100 slugs and
upstream is degraded, the worst-case wait is bounded but not visible.
**Fix**: add `httpx.AsyncClient(timeout=10.0)` for consistency.

All other `httpx.AsyncClient(...)` constructions inspected
(`crypto5min/router.py:71`, `sources/polymarket_pool.py`,
`sources/yfinance_batch.py`, terminal modules) pass an explicit timeout.

### 3.2 Retry logic on external APIs

`regression_router.py:1895-1936` shows the canonical pattern: classify
429 / 503 / 504 as transient and retry once with a fixed back-off. This
is documented in `docs/adrs/ADR-0012-rate-limit-retry.md`. The terminal
news / trades modules implement the same retry pattern
(`terminal/news.py:286, 360` etc.) with capped attempts and exponential
back-off (`0.5 * (2 ** attempt)`).

**No finding.**

### 3.3 Bare `except:` clauses

`grep -rEn "^\\s*except\\s*:" api/src/pfm/` returns **zero**. CLAUDE.md
mandate satisfied.

`except Exception` (broad) appears 316 times. Spot-check of `main.py`
sites shows every one is either:

- inside a `# pragma: no cover - defensive` block in lifespan startup,
- inside a "swallow + warn" log call,
- inside the singleflight cache layer where the policy is "never let a
  cache fault crash the request"

This is the right idiom for a public-facing service; we do **not** want
a single Polymarket 429 to crash a worker. **No finding.**

### 3.4 NaN/Inf handling in numerical paths

Spot-checks of `model.py`, `attribution.py`, `decay_monitor.py`,
`pricing/binary_models.py`:

- `pricing/binary_models.py:358, 474` ŌĆö wraps the divisor in
  `(ValueError, ZeroDivisionError)` try/except.
- `regression_router.py:211` ŌĆö wraps in `(ValueError, ZeroDivisionError,
  np.linalg.LinAlgError)`.
- `terminal/compare.py:236`, `terminal/quote.py:580` ŌĆö same pattern.

`np.linalg.LinAlgError` (singular matrix on OLS solve) is caught at the
top-level fit handler in `regression_router.py` and converted to a 422
with a meaningful detail. The OLS path uses `statsmodels` with
`cov_type='HAC'` per CLAUDE.md mandate; HAC is robust to
heteroskedasticity but **not** to NaN inputs ŌĆö verified that the data-
prep pipeline drops NaN rows before fitting (`model.py` `.dropna()` on
the joined `(y, X)` frame).

**No finding.**

### 3.5 Background-task supervision

The arb-engine autostart (`main.py:529-700`) uses a Redis SETNX leader
lock with Lua-CAS refresh and an orphan-reaper (`pgrep` + SIGTERM). Lock
extension is atomic via `_redis.eval(LUA_REFRESH, ...)`. This is correct
distributed-systems hygiene; the failure mode (Redis unavailable) is
logged and the engine is simply not started. **Good design.**
